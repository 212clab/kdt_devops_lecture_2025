# LAMP Stack with Docker Compose

## 🎯 프로젝트 개요
Docker Compose를 활용하여 구축한 완전한 LAMP 스택 개발 환경

### 기술 스택
- **웹서버**: Nginx (Alpine)
- **애플리케이션**: PHP 8.2 FPM
- **데이터베이스**: MySQL 8.0
- **관리도구**: phpMyAdmin
- **오케스트레이션**: Docker Compose

## 🚀 주요 기능
- 완전한 웹 개발 환경 구축
- 실시간 코드 수정 및 반영
- 데이터베이스 연동 웹 애플리케이션
- 팀 협업 개발 환경

## 📊 실습 성과
### 기술적 성과
- Docker Compose 고급 기능 활용 (의존성, 헬스체크, 환경변수)
- 컨테이너 간 네트워킹 구성 (nginx → php → mysql)
- 볼륨 마운트를 통한 개발 워크플로우 구축
- 실행 중인 컨테이너 직접 조작 경험

### 협업 성과
- 팀별 기능 개발 및 통합
- 동일한 개발 환경에서의 협업 경험
- 컨테이너 기반 개발의 장점 체험

## 🛠️ 사용법
```bash
# 환경 구축
git clone [repository]
cd lamp-stack

# 서비스 실행
docker-compose up -d

# 접속 확인
# http://localhost - 메인 애플리케이션
# http://localhost:8080 - phpMyAdmin
```

## 🔧 컨테이너 조작 경험
### PHP 컨테이너 접근
```bash
docker exec -it lamp-php sh
vi /var/www/html/index.php  # 실시간 수정
```

### MySQL 데이터 조작
```bash
docker exec -it lamp-mysql mysql -u lampuser -p
USE lampdb;
SELECT * FROM guestbook;
```

# LAMP 스택 실습 기술 인사이트

## 💡 핵심 학습 포인트

### 1. Docker Compose의 실무적 가치
- **환경 일관성**: 모든 개발자가 동일한 환경에서 작업
- **빠른 환경 구축**: `docker-compose up` 한 번으로 완전한 개발 환경
- **격리된 개발**: 호스트 시스템에 영향 없는 독립적 환경

### 2. 컨테이너 네트워킹의 이해
- **서비스 디스커버리**: 컨테이너명으로 직접 통신 (nginx → php, php → mysql)
- **포트 매핑**: 내부 포트와 외부 포트의 분리
- **네트워크 격리**: 전용 네트워크를 통한 보안 강화

### 3. 실시간 개발 워크플로우
- **볼륨 마운트**: 소스 코드 실시간 반영
- **컨테이너 접근**: 운영 중인 시스템 직접 조작
- **로그 모니터링**: `docker-compose logs`를 통한 실시간 디버깅

## 🔍 실무 적용 시나리오

### 개발팀 협업
- 모든 개발자가 동일한 Docker Compose 환경 사용
- 환경 설정 차이로 인한 "내 컴퓨터에서는 잘 되는데" 문제 해결
- 새로운 팀원의 빠른 온보딩

### 스테이징 환경
- 프로덕션과 유사한 환경에서 테스트
- 환경변수를 통한 설정 분리
- 데이터베이스 마이그레이션 테스트

## ⚠️ 주의사항 및 한계
- **성능**: 개발 환경 최적화, 프로덕션에는 추가 고려사항 필요
- **보안**: 개발용 설정이므로 프로덕션에서는 보안 강화 필수
- **데이터 백업**: 볼륨 데이터의 백업과 복구 전략 필요

## 🚀 다음 단계 학습 계획
1. **보안 강화**: 컨테이너 보안 스캔과 정책 적용
2. **성능 최적화**: 이미지 크기 최적화와 리소스 관리
3. **모니터링**: 로그 수집과 메트릭 모니터링 시스템
